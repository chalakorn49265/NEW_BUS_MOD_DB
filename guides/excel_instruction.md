Create an Excel workbook in a professional consulting style suitable for Deloitte / EY / strategy-client presentation standards.

OBJECTIVE
Build a client-facing Excel file that is clean, executive-ready, easy to navigate, and visually polished. The workbook should look like something used in consulting or transaction advisory presentations, not a raw analyst file.

OVERALL STYLE

- Professional, minimal, corporate
- Clean white background
- Use a restrained color palette: dark blue, teal, gray, black
- Avoid bright colors, clutter, and excessive borders
- Prioritize readability, structure, and executive-level presentation
- The workbook should feel suitable for senior management or client meetings

WORKBOOK STRUCTURE
Create the workbook with these types of sheets where relevant:

1. Cover / Dashboard
2. Executive Summary
3. Key Assumptions / Inputs
4. Analysis / Model
5. Output Tables
6. Charts / Visual Summary
7. Appendix / Backup Calculations

FORMATTING REQUIREMENTS

1. Title and Header Style

- Each sheet should have a clear title at the top
- Add a professional header area with:
  - Project / client name
  - Topic / analysis name
  - Date / version
- Sheet titles should be bold, larger font, and in dark blue or dark gray
- Freeze top row where useful

2. Font and Layout

- Use a professional font such as Aptos, Arial, or Calibri
- Keep font sizes consistent:
  - Main title: 16–20 pt
  - Section headers: 11–13 pt bold
  - Body cells: 9–11 pt
- Align numbers neatly
- Use consistent spacing between sections
- Avoid overcrowding

3. Color Logic
   Use consistent formatting logic:

- Dark blue / teal for section headers
- Light gray or very light blue fill for input areas
- White or light neutral fill for outputs
- Soft highlight color for key takeaway metrics
- Red/orange only for warnings or negative results
- Green only for positive performance or validated outputs
  Do not overuse colors

4. Input / Calculation / Output Separation
   Clearly distinguish:

- Inputs
- Calculations
- Final outputs
  Suggested convention:
- Inputs = light fill + bordered box
- Calculations = plain white cells
- Outputs / KPIs = bold, boxed, highlighted
  This separation must be obvious to a client or reviewer

5. Number Formatting

- Use comma separators
- Keep decimal places consistent
- Use percentages properly formatted
- Use currency formatting where relevant
- Use units clearly in headers or row labels
- Avoid inconsistent precision across comparable metrics

6. Borders and Gridlines

- Remove default clutter where possible
- Use subtle borders only to organize key sections
- Avoid thick borders everywhere
- Make tables look polished and presentation-ready

7. Charts and Visuals
   Where charts are included:

- Make them simple, clean, and boardroom-friendly
- No unnecessary 3D effects
- Use consistent corporate colors
- Add clear chart titles
- Label axes properly
- Use minimal but sufficient legends
- Emphasize the story, not decoration

8. Dashboard Expectations
   If there is a dashboard sheet, it should contain:

- Title and short subtitle
- 3–6 key KPI boxes at top
- One or two major charts
- Small takeaway section or insight box
- Clean layout with balanced spacing
  Dashboard must feel presentation-ready and suitable for screenshots into PowerPoint

9. Executive Summary Expectations
   This sheet should summarize:

- Key objective
- Main assumptions
- Key outputs / recommendations
- Main risks / sensitivities
  Use concise tables and highlighted takeaway boxes

10. Assumptions Sheet

- Organize assumptions by category
- Keep labels clear and business-friendly
- Group related assumptions together
- Make changeable cells obvious
- Include units and comments where helpful

11. Analysis Sheets

- Organize calculations in logical blocks
- Add section headers
- Keep formulas structured and traceable
- Avoid messy or hidden logic
- Make it easy for another analyst to audit

12. Output Tables

- Build polished summary tables
- Put the most important metrics first
- Order rows and columns logically
- Use spacing and borders to improve readability
- Tables should be suitable for direct client viewing

13. Appendix / Backup

- Put detailed calculations or supporting tables here
- Keep main client-facing sheets clean
- Appendix can be more detailed but still organized

CONSULTING PRESENTATION FEEL
The workbook should feel like:

- Client-ready
- Structured like a consulting deliverable
- Easy for executives to scan
- Visually disciplined
- Suitable to present in a meeting without embarrassment

QUALITY CONTROL
Before finalizing:

- Check alignment consistency
- Check row heights / column widths
- Ensure no overlapping text
- Ensure all sheet names are professional
- Ensure key outputs stand out clearly
- Ensure there are no random colors or inconsistent formats
- Ensure the workbook feels premium and polished

DELIVERABLE STANDARD
Output should resemble a workbook prepared by a Big Four consulting / advisory team for client presentation:

- polished
- executive-friendly
- visually clean
- logically structured
- presentation-ready

IMPORTANT
Do not make it look like a basic spreadsheet or academic homework file.
Make it look like a real consulting deliverable prepared for a client presentation.
